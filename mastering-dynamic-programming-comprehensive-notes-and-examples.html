Longest Common Subsequence

Definition: A subsequence of a given sequence is just the give sequence with zero or more elements left out. Formally, given a sequence $X=[x_1,x_2,...,x_m]$, another sequence $Z=[z_1,z_2,...,z_k]$ is a subsequence of $X$ if there exists a strictly increasing sequence $[i_1,i_2,...,i_k]$ of indices of $X$ such that for all $j=1,2,...,k$, we have $x_{i_j}=z_j$. For example, $Z=[B,C,D,B]$ is a subsequence of $X=[A,B,C,B,D,A,B]$ with corresponding index sequence $[2,3,5,7]$.[^1]

In the longest-common-subsequence problem, we are given two sequences $X=[x_1,x_2,...,x_m]$ and $Y=[y_1,y_2,...,_n]$ and wish to find a maximum-length common subsequence of $X$ and $Y$.

We can define a table, $c[m+1,n+1]$, where each element $c[i,j]$ represents the length of an LCS of the prefix sequences $X_{i-1}$ and $Y_{j-1}$. Here $X_{i-1}$ refers to the substring of $X$ that starts from index $0$ and ends at index $i-1$, and $Y_{j-1}$ refers to the substring of $Y$ that starts from index $0$ and ends at index $j-1$.

Theorem 15.1(Optimal substructure of an LCS)[^2] Let $X=[x_1,x_2,...,x_m]$ and $Y=[y_1,y_2,...,_n]$ be sequences, and let $Z=[z_1,z_2,...,z_k]$ be any LCS of $X$ and $Y$.

  1. If $x_m=y_n$, then $z_k=x_m=y_n$ and $Z_{k-1}$ is an LCS of $X_{m-1}$ and $Y_{n-1}$.
  2. If $x_m\neq y_n$, then $z_k\neq x_m$ implies that $Z$ is an LCS of $X_{m-1}$ and $Y$.
  3. If $x_m\neq y_n$, then $z_k\neq y_n$ implies that $Z$ is an LCS of $X$ and $Y_{n-1}$.

If either $i=0$ or $j=0$, one of the sequences has length $0$, and so the LCS has length $0$. The optimal substructure of the LCS problem gives the recursive formula

$$ c[i,j]=\begin{cases} 0&i=0\lor j=0\\ c[i-1,j-1]+1&i,j>0 \land x_{i-1}=y_{j-1}\\ \max(c[i-1,j],c[i,j-1])& i,j>0\land x_{i-1}\neq y_{j-1} \end{cases} $$

Based on the above equation, we could easily write a bottom-up dynamic programming to compute the solutions. Function lcs-length takes two Strings as inputs. It stores the $c[i,j]$ values in a table c[m+1,n+1].0 and it also maintains the direction which is stored in c[m+1,n+1].1 to help us construct an optimal solution.

import Foundation
enum Direction {
case north_west /* ↖ */
case left /* ↑ */
case up /* ← */
case none /* · */
}
func lcs_length(_ x : String, _ y : String) {
let X = Array(x)
let Y = Array(y)
let m = X.count
let n = Y.count
var c = [[(Int, Direction)]](
repeating: [(Int, Direction)](
repeating: (0, .none),
count: n + 1
),
count: m + 1
)
for i in 1 ... m {
for j in 1 ... n {
if X[i - 1] == Y[j - 1] { /* Go north west */
c[i][j].0 = c[i - 1][j - 1].0 + 1
c[i][j].1 = .north_west
} else {
if c[i - 1][j].0 >= c[i][j - 1].0 {
c[i][j].0 = c[i - 1][j].0
c[i][j].1 = .up
} else {
c[i][j].0 = c[i][j - 1].0
c[i][j].1 = .left
}
}
}
}
/* Print the c table (only try this with small strings) */
var is_first = true
for i in c.indices {
var separator = "+-+-+"
var header_num = "| |j|"
var header = "|i| |"
var line1 = "|\(String(i, radix: 16))| |"
var line2 = "| |\(i == 0 ? " " : X[i - 1])|"
for j in c[i].indices {
if c[i][j].1 == .none {
line1 += "··"
line2 += "·\(c[i][j].0)"
} else if c[i][j].1 == .left {
line1 += "··"
line2 += "←\(c[i][j].0)"
} else if c[i][j].1 == .up {
line1 += "·↑"
line2 += "·\(c[i][j].0)"
} else {
line1 += "↖·"
line2 += "·\(c[i][j].0)"
}
line1 += "|"
line2 += "|"
separator += "--+"
header += " \(j == 0 ? " " : Y[j - 1])|"
header_num += "\(String(j, radix: 16)) |"
}
if is_first {
print(separator)
print(header_num)
print(separator)
print(header)
print(separator)
is_first = false
}
print(line1)
print(line2)
print(separator)
}
}
lcs_length("HIEROGLYPHOLOGY", "MICHAELANGELO")

Longest (Monotonically) Increasing Subsequence

In this problem, we are given a sequence of numbers or characters $X=[x_1,x_2,...,x_n]$ and wish to find a longest subsequence of $X$ that strictly increases (for characters: lexicographically).

We can define a table $c[n]$, where each element $c[i]$ represents the length of an LIS of the prefix sequence $X_i$. Here $X_i$ refers to the substring of $X$ that starts from index $0$ and ends at index $i$.

We can initialize $c[0]=1$ since any individual element is a subsequence of length 1. Then the next question is, what is the second element of the LIS? We have two choices: either extend a previous LIS (and take the maximum) by $1$ or don't extend any of them and let $c[1]=1$. For the remaining LIS, we adopt the same method. Now we have the dynamic formula

$$ c[i]=\begin{cases}\max\{c[j]|0\leq j<i\land X[i]>X[j]\}+1&\text{if such a set exists}\\1&\text{otherwise}\end{cases} $$

Based on the above equation, we could easily write a bottom-up dynamic programming to compute the solutions. Function lis-length take a String as its input and returns the length of an LIS. In this example, it will return $5$, which is the length of the LIS ABORT.

import Foundation
func lis_length(_ x : String) -> Int {
let X = Array(x)
let n = X.count
var c = [Int](repeating: 1, count: n)
for i in 0 ..< n {
for j in 0 ..< i {
if X[i] > X[j] {
c[i] = max(c[i], c[j] + 1)
}
}
}
return c.max()!
}
print(lis_length("CARBOHYDRATE"))

The running time of the function is $\Theta(n^2)$, since each subproblem takes $\Theta(n)$ time to compute and we have $\Theta(n)$ subproblems.

All-Pairs Shortest Paths

In this section, we shall use a dynamic-programming formulation to solve the all-pairs shortest paths problem on a directed graph $G=(V,E)$. The resulting algorithm, known as the Floyd-Warshall algorithm, runs in $\Theta(n^3)$ time.

In this problem, we wish to find, for every pair of vertices $u,v\in V$, a shortest (least-weight) path from $u$ to $v$, where the weight of a path is the sum of the weights of its constituent edges.

We can define the subproblem $\delta_k(u,v)$ as the minimum weight of path from $u$ to $v$ that only uses vertices from $\{1,2,...,k\}\cup\{u,v\}$ (By convention we number vertices so that $V=\{1,2,...,|V|\}$). When $k=0$, a path from vertex $i$ to vertex $j$ with no intermediate vertex numbered higher that $0$ has no intermediate vertices at all. Such a path has at most one edge, and hence $\delta_0(i,j)=w_{ij}$. Following the above discussion, we define $\delta_k(i,j)$ recursively by

$$ \delta_k(i,j)=\begin{cases}w_{ij}&k=0\\\min(\delta_{k-1}(i,j),\delta_{k-1}(i,k) + \delta_{k-1}(k,j))&k\geq1\end{cases} $$

Based on the above recurrence, we can use the following bottom-up procedure to compute $\delta_k(i,j)$ in order of increasing values of $k$. Its input is an $n\times n$ matrix $W$ defined as

$$ w_{ij}=\begin{cases}0&i=j\\\text{the weight of directed edge }(i,j)&i\neq j\land (i,j)\in E\\\infty&i\neq j\land (i,j)\notin E\end{cases} $$

func floyd_warshall(_ w : [[Int]]) -> [[Int]] {
var d = w
let n = w.count
for k in 0 ..< n {
for i in d.indices {
for j in d[i].indices {
d[i][j] = min(d[i][j], d[i][k] + d[k][j])
}
}
}
return d
}

The running time of the Floyd-Warshall algorithm is determined by the triply nested for loops. Because each subproblem takes $O(1)$ time, the algorithm runs in time $\Theta(n^3)$.
